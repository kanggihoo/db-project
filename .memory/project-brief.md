# Project Brief

## 프로젝트 목표

이 프로젝트는 이커머스 도메인을 대상으로 DB 성능 최적화를 단계적으로 학습하고 증명하는 실험용 Spring Boot 프로젝트다.

핵심 목표는 최적화 없는 나이브한 구현에서 출발해, 각 Phase에서 발생하는 문제를 k6, Grafana, Prometheus, PostgreSQL 실행계획으로 수치화하고 Before/After를 비교하는 것이다.

## 장기 로드맵

- Phase 0: 인프라, 스키마, 더미 데이터 시딩 기반 구축
- Phase 1: 나이브 API 구현과 베이스라인 확보
- Phase 2: 인덱스 설계와 `EXPLAIN ANALYZE` 기반 실행계획 분석
- Phase 3: N+1 재현과 로딩 전략 최적화
- Phase 4: 트랜잭션 격리 수준과 동시성 실험
- Phase 5: QueryDSL, DTO Projection, 벌크 연산 기반 쿼리 최적화
- Phase 6: 집계 쿼리와 인덱스 최적화
- Phase 7: Offset 페이지네이션 한계와 Cursor 페이지네이션 전환

## 핵심 사용자와 사용 맥락

- 주 사용자: DB 성능 최적화, Spring Boot/JPA, PostgreSQL 실행계획을 학습하는 개발자
- 사용 맥락: 로컬 Docker Compose 환경에서 대용량 더미 데이터를 넣고 API, 부하 테스트, 모니터링을 통해 병목을 직접 확인한다.
- 산출물 기준: 단순 구현 완료가 아니라 수치, 스크린샷, 실행계획, 테스트 결과로 학습 포인트를 증명해야 한다.

## 제품적 경계

- 실제 쇼핑몰 서비스가 아니라 DB 최적화 학습과 재현 가능한 실험이 목적이다.
- Phase별 문제를 의도적으로 드러내기 위해 일부 비효율은 설계상 유지한다.
- 최적화는 한 번에 모두 적용하지 않고 RoadMap의 Phase 순서에 맞춰 도입한다.

## 장기 우선순위

1. 실험 재현성: 데이터 규모, 부하 조건, 설정값을 고정한다.
2. 정량 증명: k6 p95, TPS, 에러율, DB 지표, 실행계획을 기록한다.
3. Phase 경계 준수: 다음 Phase의 최적화를 앞당겨 적용하지 않는다.
4. 학습 가치: 병목 원인과 개선 효과가 코드와 문서에서 명확히 드러나야 한다.
