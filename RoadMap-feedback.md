
미흡한 부분
1. Phase 간 전환 기준 모호

각 Phase의 "완료 조건(Done Criteria)"이 없음. Phase 1→2로 언제 넘어가는지 수치 기준이 필요.
개선: 각 Phase 말미에 ✅ 완료 조건 섹션 추가 (예: "주문 API p95 > 500ms 확인 + BASELINE.md 작성 완료").
2. 환경/데이터 스펙 부재

더미 데이터 "100만 건 이상"이라고만 적혀있고, 테이블별 목표 row 수가 명시 안 됨.
개선: USER 10만, PRODUCT 5만, ORDER 100만, POINT_HISTORY 500만 식으로 테이블별 기준 row 수 명시. Phase별 실험이 의미 있으려면 데이터 규모가 고정돼야 함.
3. Phase 4 위치의 논리적 어색함

"쿼리 수 줄였더니 정합성 문제 발생"이라는 흐름은 자연스럽지 않음. 트랜잭션 격리는 N+1과 독립적 주제.
개선: Phase 4를 "동시성 + 락(비관/낙관락) + 격리 수준" 묶음으로 확장하거나, Phase 2 뒤로 이동 검토. 또는 전환 서사를 "재고 차감 로직을 추가하려니 동시성 문제 발견"으로 수정.
4. 누락된 주요 주제

커넥션 풀 튜닝 (HikariCP): Phase 1에서 Active connection 지표는 보는데, 튜닝 Phase가 없음.
캐싱 전략 (Redis, 2차 캐시): 로드맵에서 완전히 누락. 이커머스 특성상 상품 조회 캐싱은 필수 주제.
비관적 락 vs 낙관적 락: Phase 4에서 격리 수준만 다루고 명시적 락 전략 부재. SKU 재고 차감은 락 실험의 교과서적 소재.
개선: Phase 4에 락 실험 추가, Phase 8(캐싱) 또는 Phase 5 뒤에 캐싱 Phase 신설 고려.
5. Before/After 정량 템플릿 부재

"정량적으로 증명"이 핵심 철학인데, 각 Phase 결과 기록용 표준 템플릿이 없음.
개선: BASELINE.md, PHASE2_RESULT.md 같은 산출물의 표 포맷을 로드맵에 예시로 박아두기.
